各腳本詳細診斷與漏洞分析

Inital-setup/initial-setup.sh
RHEL 9 首次開機互動式網路與 Hostname 設定腳本
Critical
🔴 1. 完全缺乏輸入格式驗證
未對 IP、Netmask、Gateway、DNS 進行正則與數值範圍檢查，非法輸入會直接導至 NetworkManager 崩潰。
🔴 2. nmcli 位置參數錯位 Bug
當 Gateway 或 DNS 留空時，帶入 nmcli connection modify 會造成後續參數位置偏移對位失敗。
🟡 3. 缺乏多終端併發執行鎖 (Multi-TTY Race Condition)
若有多個 root console 併發登入，會同時觸發設定精靈並產生死結競爭。
Seal_RHEL_VM/seal-rhel-template.sh
RHEL VM 範本封裝與機號清理腳本
Critical
🔴 1. 強制覆寫抹除 /etc/hosts
cat > /etc/hosts 會直接刪除管理員預先設定好的自訂域名解析條目。
🔴 2. Machine-ID 寫入非法字串
寫入 'uninitialized\n' 違背 systemd 32-char 規格，會導致複製出的所有 VM 具有相同的 ID。
🟡 3. OS 偵測遺漏 AlmaLinux
Case 條件中缺少 almalinux，導致在 AlmaLinux 8/9 系統上會直接中斷退出。
pve_link_iso/Old/pve_link_iso_old.sh
舊版 Proxmox VE ISO 自動符號連結腳本
Critical
🔴 1. 管道 Subshell 變數作用域死結 Bug
將 find 透過 | while read 送入子 Shell，迴圈內累加的 COUNT 在退出後失效，導致印出的統計永遠為 0。
pve_link_iso/pve_link_iso.sh
現行版 Proxmox VE ISO 自動符號連結腳本
Medium
🟡 1. 無效軟連結 (Dangling Symlinks) 判斷 Bug
[ ! -e "$target_file" ] 對於目標已消失的軟連結會回傳 true，但執行 ln -s 會因檔案路徑已存在而失敗。

Kubernetes 節點前置環境初始化與驗證腳本

架構與系統優化建議指導方針

💡 六大企業級自動化維運守則

  1. 1. 強制輸入驗證與防呆： 所有接受人機互動或外部變數輸入的腳本，必須經過嚴格正則表達式審查 (IPv4, CIDR, DNS, Hostname)，禁止直接帶入 Shell 命令。
  2. 2. 原子性互斥鎖 (Atomic Locking)： 在開機初始化或寫入系統層級配置的腳本中，一律引進 mkdir /run/*.lock 互斥鎖，避免多重登入併發破壞系統狀態。
  3. 3. 檔案精準修剪替代全檔覆寫： 禁止使用 cat > /etc/hosts 等抹除式命令。修改配置應採用 sed 精準替換標的字串，完整保護管理員原本之自訂規則。
  4. 4. 遵循 systemd 規範處理 Machine-ID： 重置 Machine-ID 應直接刪除檔案或填入 0 位元組空檔，由 systemd 於開機時自動動態生成全新 32 位元 16 進位隨機機號。
  5. 5. 免除子 Shell (Subshell) 管道作用域陷阱： 在 Bash 統計計數或迴圈中，改用進程替換 while read ... done < <(find ...)，確保變數異動能正確回傳至父層 Shell。
  6. 6. 容器配置具備高彈性與明確標籤： 避免在 Docker Compose 中使用 latest / main 浮動標籤，並且將硬體 GPU 需求設定為彈性開關。

重構版 initial-setup.sh 範例預覽

完整的修正原始碼已整理至 docs/04_fixes.md 中，以下為重點驗證與互斥鎖實作：

# 1. 原子互斥鎖防護 LOCK_DIR="/run/initial-setup.lock" if ! mkdir "$LOCK_DIR" 2>/dev/null; then echo "[WARN] Another instance of initial-setup is running. Exiting." exit 0 fi trap 'rm -rf "$LOCK_DIR"' EXIT # 2. 純 Bash 位元運算轉換 CIDR (免 bc 依賴) mask_to_cidr() { local mask=$1 local IFS=. local -a octets read -r -a octets <<< "$mask" local bin="" for octet in "${octets[@]}"; do local val=$octet b="" for ((i=0; i<8; i++)); do b=$((val % 2))$b val=$((val / 2)) done bin="$bin$b" done local cidr=${bin%%0*} echo "${#cidr}" }
